# Text Format Converter & Formatter

## Purpose

The Text Format Converter & Formatter tool provides explicit, production-style conversion and formatting between:

- HTML
- XML
- Markdown
- RTF
- PDF
- DOCX
- JSON

The tool is designed for readable, text-focused output rather than impossible semantic round-tripping between fundamentally different formats.

## Supported Directions

Direct or structure-aware conversions:

- `JSON <-> XML`
- `HTML <-> Markdown`
- `HTML -> XML` as XHTML-like normalized output
- `Markdown -> XML` through Markdown-to-HTML then XHTML-like normalization
- `JSON/XML -> HTML` as formatted preformatted-code rendering
- `JSON/XML -> Markdown` as fenced code-block rendering

Best-effort document conversions:

- `HTML/Markdown/JSON/XML/RTF/DOCX/PDF -> RTF/DOCX/PDF`
- `RTF -> HTML/Markdown/PDF/DOCX`
- `DOCX -> HTML/Markdown/RTF/PDF`
- `PDF -> HTML/Markdown/RTF/DOCX`

Blocked conversions:

- document-to-structured directions such as `PDF/DOCX/RTF -> JSON/XML`

## Formatting Behavior

- Same-format operations are supported as format-only actions.
- `JSON`, `XML`, `HTML`, and `Markdown` are beautified or normalized directly.
- `RTF`, `DOCX`, and `PDF` are regenerated as readable text-focused outputs with warnings when fidelity is limited.

## Input and Validation

- Direct text input is limited to `5,000` characters.
- File input supports larger content but rejects:
  - files larger than `10 MB`
  - extracted content above `100,000` characters
- When both input modes are populated, loaded file input takes precedence until cleared.

## Preview Flow

- Structured text formats use a syntax-oriented `FlowDocument`.
- Document-family targets use a document-style paragraph preview.
- The preview pipeline is shared by the in-tool result pane and the modeless pop-out preview window.

## Export Flow

- Save operations are routed through `ITextResultExportService`.
- Text targets are written as UTF-8 without BOM.
- `RTF`, `DOCX`, and `PDF` exports use generated byte content returned by the conversion service.
- Save dialogs and overwrite confirmation are delegated to `IFileDialogService`.

## Known Limitations

- PDF, DOCX, and RTF conversions prioritize readable paragraph structure over original layout fidelity.
- HTML to XML output is XHTML-like and may differ from browser-emitted markup.
- XML to JSON uses `@attribute` and `#text` keys when attributes or mixed content are present.
- JSON to XML generates a synthetic `<root>` wrapper and `<item>` elements for arrays.

## Extending the Tool

To add a new format or improve fidelity:

1. Extend `TextFormatKind` metadata and file-extension mappings.
2. Update the conversion matrix in `TextFormatConversionService.GetConversionSupport`.
3. Add conversion and formatting behavior in `TextFormatConversionService`.
4. Update preview behavior in `TextPreviewDocumentBuilder`.
5. Add or adjust export logic if the new format needs generated binary output.
6. Cover the new path with service and view-model tests.
