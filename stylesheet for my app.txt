•	CheckBox → default chrome checkmarks
•	ToolTip → default yellow/white system tooltip
4. No Animations
Zero storyboards or visual state transitions. Buttons snap between states, dropdowns appear/disappear abruptly, theme switching is jarring.

<ResourceDictionary xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
                    xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml">

    <!-- Thin modern scrollbar thumb -->
    <Style x:Key="ScrollBarThumbStyle" TargetType="Thumb">
        <Setter Property="OverridesDefaultStyle" Value="True"/>
        <Setter Property="IsTabStop" Value="False"/>
        <Setter Property="Template">
            <Setter.Value>
                <ControlTemplate TargetType="Thumb">
                    <Border x:Name="ThumbBorder"
                            Background="{DynamicResource ScrollBarThumbBrush}"
                            CornerRadius="3"
                            Margin="1"/>
                    <ControlTemplate.Triggers>
                        <Trigger Property="IsMouseOver" Value="True">
                            <Setter TargetName="ThumbBorder" Property="Background"
                                    Value="{DynamicResource ScrollBarThumbHoverBrush}"/>
                        </Trigger>
                    </ControlTemplate.Triggers>
                </ControlTemplate>
            </Setter.Value>
        </Setter>
    </Style>

    <!-- Vertical scrollbar: 6px wide, no arrows -->
    <ControlTemplate x:Key="VerticalScrollBarTemplate" TargetType="ScrollBar">
        <Grid Width="6">
            <Border Background="{DynamicResource ScrollBarTrackBrush}" CornerRadius="3"/>
            <Track x:Name="PART_Track" IsDirectionReversed="True">
                <Track.Thumb>
                    <Thumb Style="{StaticResource ScrollBarThumbStyle}"/>
                </Track.Thumb>
            </Track>
        </Grid>
    </ControlTemplate>

    <!-- Horizontal scrollbar: 6px tall, no arrows -->
    <ControlTemplate x:Key="HorizontalScrollBarTemplate" TargetType="ScrollBar">
        <Grid Height="6">
            <Border Background="{DynamicResource ScrollBarTrackBrush}" CornerRadius="3"/>
            <Track x:Name="PART_Track" IsDirectionReversed="False">
                <Track.Thumb>
                    <Thumb Style="{StaticResource ScrollBarThumbStyle}"/>
                </Track.Thumb>
            </Track>
        </Grid>
    </ControlTemplate>

    <!-- Apply globally -->
    <Style TargetType="ScrollBar">
        <Setter Property="OverridesDefaultStyle" Value="True"/>
        <Setter Property="SnapsToDevicePixels" Value="True"/>
        <Style.Triggers>
            <Trigger Property="Orientation" Value="Vertical">
                <Setter Property="Width" Value="6"/>
                <Setter Property="Template" Value="{StaticResource VerticalScrollBarTemplate}"/>
            </Trigger>
            <Trigger Property="Orientation" Value="Horizontal">
                <Setter Property="Height" Value="6"/>
                <Setter Property="Template" Value="{StaticResource HorizontalScrollBarTemplate}"/>
            </Trigger>
        </Style.Triggers>
    </Style>

</ResourceDictionary>

Ask mode is fine
Since this is a large-scale visual rework touching many files, "Ask" mode gives you control to review each change before it lands. You can always switch to "Agent" later for smaller follow-up tasks.
Theme toggle + Settings menu — my recommendation
Keep the theme toggle in the header. It's a 1-click action users reach for constantly — burying it behind a settings menu adds friction. Windows 11 itself keeps the theme toggle at the top level in Quick Settings, not buried in the Settings app.
But yes, add a Settings popup window. Here's what belongs there:
Setting	Why
Default theme preference (+ "System" option)	Persist choice, auto-follow Windows theme
Startup page (Home vs. last-used tool)	Power user convenience
Window behavior (remember size/position)	Fixes your DPI/scaling complaint — user picks what works
Reset all tools to defaults	Housekeeping
About / version info	Standard
The header would then have two buttons in the top-right: the theme toggle (☀/🌙) and a ⚙ gear icon that opens the settings Window.
A popup Window (not a Page navigation) is the right call — it won't interfere with tool state, and ShowDialog() keeps it modal so there's no confusion about which window is active.
I'll build the settings window as part of Phase 1 alongside the theme rework since they're coupled (persisting theme choice, system theme detection).
